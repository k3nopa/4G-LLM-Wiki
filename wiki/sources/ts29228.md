# Source: 3GPP TS 29.228 v18.0.0

**Full title:** IP Multimedia (IM) Subsystem Cx and Dx Interfaces; Signalling flows and message contents  
**Version:** 18.0.0 (Release 18, 2024-05)  
**File:** `raw/papers/ts_129228v180000p.pdf`  
**Companion spec:** TS 29.229 (Diameter protocol/ABNF details — see [Cx/Dx Diameter Protocol](../protocols/Cx-Diameter.md))

## Scope

Specifies:
1. Interactions between HSS and CSCF (Cx interface) — signalling flows and message parameter tables
2. Interactions between CSCF and SLF (Dx interface)
3. Cx interface for Mission Critical Services (TS 23.280) where "SIP Core" = CSCF and "SIP database" = HSS

Also addresses Px interface (Presentity Presence Proxy ↔ HSS via Cx mechanisms) and SLF/DPA resolution.

## Key Definitions (§3.1)

| Term | Definition |
|---|---|
| Common Part (user profile) | iFC instances evaluated for BOTH registered and unregistered PUIs, or unregistered PSIs |
| Complete user profile | All three parts: registered + unregistered + common |
| Registered Part | iFCs evaluated only for registered PUIs; not evaluated when PUI is unregistered |
| Unregistered Part | iFCs evaluated only for unregistered PUIs; not evaluated when PUI is registered |
| Unregistered State | PUI not registered but has S-CSCF assigned for unregistered-state services |
| Not Registered State | PUI not registered and NO S-CSCF assigned |
| Authentication pending flag | Flag indicating authentication of a PUI-IMPI pair is pending confirmation |
| S-CSCF reassignment pending flag | Indicates subscription may be reassigned to new S-CSCF (current S-CSCF not responding) |
| User information | User profile + charging info + allowed WAF/WWSF identities + authentication info |
| Allowed WAF/WWSF identities | Network addresses of WebRTC Authentication Functions / WebRTC Web Server Functions |
