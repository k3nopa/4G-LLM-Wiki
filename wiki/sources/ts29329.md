# 3GPP TS 29.329 v16.2.0 — Sh Interface Based on Diameter Protocol

**Full title:** Sh interface based on the Diameter protocol; Protocol details (Release 16, 2021-04)
**PDF:** `raw/papers/ts_129329v160200p.pdf`
**Total pages:** ~28 (+ change history)

## Scope

Defines the Diameter wire protocol for the **Sh reference point** between:
- Application Servers (AS) or Subscription Correlated Servers (SCS) and the HSS

Companion to TS 29.328 (Sh signalling flows and message contents).

## Chunk ts29329-1 — §1–§6.2: Commands and Result Codes

### §1 Scope
Sh interface protocol for IMS CN based on IETF RFC 6733. Covers AS↔HSS and SCS↔HSS.

### §4–§5 General / Base Protocol Usage
- Diameter base protocol: IETF RFC 6733; TS 29.229 §5 clarifications apply to Sh
- Vendor-specific application: 3GPP Vendor-ID = 10415, Application-ID = 16777217

### §6.1 Command-Code Values

| Command | Code | Direction |
|---|---|---|
| UDR / UDA | 306 | AS→HSS / HSS→AS |
| PUR / PUA | 307 | AS→HSS / HSS→AS |
| SNR / SNA | 308 | AS→HSS / HSS→AS |
| PNR / PNA | 309 | HSS→AS (push) / AS→HSS |

Full ABNF message formats for all 8 commands ingested. See [protocols/Sh-Diameter.md](../protocols/Sh-Diameter.md).

### §6.2 Result-Code AVP Values

**Permanent failures (5xxx, in Experimental-Result):**
5100 USER_DATA_NOT_RECOGNIZED, 5101 OPERATION_NOT_ALLOWED, 5102 USER_DATA_CANNOT_BE_READ, 5103 USER_DATA_CANNOT_BE_MODIFIED, 5104 USER_DATA_CANNOT_BE_NOTIFIED, 5008 TOO_MUCH_DATA, 5105 TRANSPARENT_DATA_OUT_OF_SYNC, 5011 FEATURE_UNSUPPORTED, 5106 SUBS_DATA_ABSENT, 5107 NO_SUBSCRIPTION_TO_DATA, 5108 DSAI_NOT_AVAILABLE, 5002 IDENTITIES_DONT_MATCH.

**Transient failures (4xxx):**
4100 USER_DATA_NOT_AVAILABLE, 4101 PRIOR_UPDATE_IN_PROGRESS.

### §6.4 Namespace Assignments
- AVP codes: 3GPP vendor namespace
- Experimental-Result-Code: 4100–4101 (transient), 5100–5105 (permanent)
- Command codes: 306–309 (IANA per RFC 3589)
- Application-ID: 16777217 (IANA)

### §7 Version Control
Retransmit UDR/PUR/SNR with same Session-Id on timeout. Repository data concurrency via Sequence-Number.
