# 3GPP TS 29.165 v16.6.0 — Inter-IMS Network to Network Interface (NNI)

**Full title:** Inter-IMS Network to Network Interface (NNI)
**Version:** 16.6.0 (Release 16, 2021-08)
**PDF:** `raw/papers/ts_129165v160600p.pdf`
**Pages:** ~167pp (main body pp 9–84; Annexes A–D pp 85–167)

## Scope

Addresses the II-NNI (Inter-IMS Network to Network Interface) between two IM CN subsystem networks, comprising:
- **Ici reference point**: control plane SIP signalling between IBCFs
- **Izi reference point**: user plane media streams between TrGWs

Covers: control plane SIP/SDP profiling, security, numbering/naming/addressing, user plane (codecs, transport), all IMS supplementary services, charging, ICS/SRVCC/presence/messaging/OMR/IUT interoperability over II-NNI.

**Out of scope:** Mr/Mr'/Rc interfaces between CSCF-MRFC/AS; NNI on ISC/Rc for MCPTT; NNI on ISC for enterprise PBX interworking.

## Key Concepts Extracted (Chunk 1)

### II-NNI Architecture
- Two reference points: **Ici** (SIP control, IBCF↔IBCF) and **Izi** (RTP media, TrGW↔TrGW)
- Four traversal scenarios identified by `iotl` SIP URI parameter (RFC 7549): non-roaming, loopback (LBO), roaming home-to-visited, roaming visited-to-home
- IBCF is entry/exit point, acts as B2BUA when performing IMS-ALG (IPv4/IPv6 interworking)
- TrGW performs NA(P)T-PT (IPv4/IPv6 translation) and media relay

### SIP at II-NNI
- 23 supported methods (Table 6.1); ACK/BYE/CANCEL/INVITE/OPTIONS/PRACK/UPDATE mandatory; REGISTER/NOTIFY/SUBSCRIBE mandatory only on roaming II-NNI
- 25 SIP header fields with trust/no-trust behaviour (Table 6.2); P-Asserted-Identity stripped when no trust
- 127-item capability profile (Table 6.1.3.1) with 8 conditional codes (c1–c8) mapping to roaming/non-roaming, trust/no-trust, emergency scenarios
- SDP not encrypted over II-NNI; IBCF must be able to inspect/modify SDP

### Transport
- Control plane: TCP default, SCTP optional (recommended for Ici reliability)
- User plane: RTP/UDP mandatory; TCP/SCTP optional
- SIP timers: T1=500ms, T2=4s, T4=5s; Timer C >3min

## Key Concepts Extracted (Chunk 2)

### §8 Numbering/Addressing
- SIP URI mandatory at all II-NNI types; tel URI mandatory at roaming II-NNI + loopback
- Global E.164 number format for tel URI and SIP URI with user=phone at non-roaming/loopback
- URI parameters: `sos` (roaming-mandatory), `oli`/`cpc` (per agreement), `rn`/`npdi` (number portability), `isub`, `premium-rate`
- SDP: MSRP URI required at roaming II-NNI

### §11 Charging
- P-Charging-Vector IOI types: type 1 (roaming II-NNI: visited network identities), type 2 (home II-NNI: home network identities), transit-ioi (transit scenario)
- Tariff information via `application/vnd.etsi.sci+xml` in INVITE/18x/INFO per bilateral agreement

### §12 Supplementary Services (26 services, all per bilateral agreement)
26 MMTEL supplementary services catalogued with their II-NNI SIP requirements:
OIP/OIR, TIP/TIR, ACR, CDIV, CW, HOLD, MWI, ICB/OCB, CCBS, CCNR, ECT, CAT, CRS, CUG, PNM, 3PTY, CONF, FA, Announcements, AOC, CCNL, USSD, eCNAM, MuD, MiD, MCID.

## Key Concepts Extracted (Chunk 3)

### §13–§14 ICS and Service Continuity
- ICS: `g.3gpp.ics`/`g.3gpp.accesstype` feature tags; Target-Dialog for ICS dialogs; REFER with `method=BYE/INVITE`
- SRVCC: PS-to-CS (basic/alerting/pre-alerting/ATCF/mid-call) and CS-to-PS variants; DRVCC PS-to-CS and CS-to-PS; PS-PS access transfer
- ATCF feature tags: `g.3gpp.atcf`, `g.3gpp.atcf-mgmt-uri`, `g.3gpp.atcf-path`
- SRVCC tags: `g.3gpp.srvcc`, `g.3gpp.cs2ps-srvcc`, `g.3gpp.dynamic-stn`

### §15–§18 Presence, Messaging, OMR, IUT
- Presence: PUBLISH/SUBSCRIBE/NOTIFY with `application/pidf+xml`; OMA R1.1 Suppress-If-Match; OMA R2.0 `application/vnd.oma.suppnot+xml`
- Messaging: page-mode (MESSAGE + recipient-list + im-iscomposing+xml); session-mode MSRP; conference messaging via REFER + `isfocus`
- OMR: 9 SDP attributes (`visited-realm`, `secondary-realm`, `omr-codecs`, `omr-m-att`, `omr-s-att`, `omr-m-bw`, `omr-s-bw`, `omr-s-cksum`, `omr-m-cksum`); realm names must be globally unique
- IUT: without collaborative session (REFER + Replaces/Target-Dialog); with collaborative session (`g.3gpp.iut-controller`); replication pull/push via `application/vnd.3gpp.replication+xml`

### §19–§21 LBO, MRB, Overload
- LBO: `g.3gpp.trf`, `g.3gpp.loopback` feature tags; `loopback-indication` in P-Charging-Vector
- MRB delivery: `g.3gpp.mrb` feature-capability in Feature-Caps
- Overload: RFC 7339 feedback-based (no extra spec needed) + load-filter SUBSCRIBE/NOTIFY with `application/load-control+xml`; MPS exempt

### §22–§33 + Annex A Advanced features and header summary
- Original destination identity via History-Info `mp` parameter (§22)
- Telepresence: `+sip.clue` header; SCTP-DTLS data channel (§23)
- Premium rate barring: `premium-rate` URI parameter at roaming II-NNI (§24)
- P-CSCF restoration: PCRF/PCF based (Restoration-Info + IMSI in INVITE) and HSS/UDM based (Restoration-Info in 408/504) (§25)
- Resource sharing: Resource-Share header in ACK/INVITE/PRACK/REGISTER/UPDATE + 18x/2xx (§26)
- Service access number translation: `cause=380` URI parameter (§27)
- Mission critical (§28): MCPTT/MCVideo/MCData — ICSIs, affiliation, MBMS, conference event, group regrouping, functional alias management
- STIR: Attestation-Info + Origination-Id (non-roaming II-NNI); verstat URI parameter; 607 Unwanted; `g.3gpp.verstat`/`sip.607` feature-capabilities (§29)
- Emergency: `sos` URI + `urn:service:sos` URN + NG-eCall MSD (§30)
- RLOS: `+g.3gpp.rlos` Contact parameter; `urn:ims.icsi.rlos` (§31)
- PS data off: `+g.3gpp.ps-data-off` feature tag in REGISTER Contact (§32)
- MTSI data channel: `+sip.app-subtype=webrtc-datachannel`; `a=dcmap`/`a=3GPP-qos-hint`; streams 100/110 at II-NNI (§33)
- Annex A (Table A.1): 76 SIP headers with m/o/n/a/conditional codes; 34 mandatory; 7 n/a; roaming-only conditional set; trust-conditional set; non-roaming-only set
