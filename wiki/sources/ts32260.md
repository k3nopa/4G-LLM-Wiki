# 3GPP TS 32.260 v17.3.0 — IMS Charging

**Full title:** Telecommunication management; Charging management; IP Multimedia Subsystem (IMS) charging
**Version:** 17.3.0 (Release 17, 2022-07)
**PDF:** `raw/papers/ts_132260v170300p.pdf`
**Total pages:** ~235

## Scope

TS 32.260 specifies IMS-specific charging: offline (Rf/CDR), online (Ro/credit-control), and converged (Nchf/CHF) charging applied to IMS network elements. It maps the general 3GPP charging architecture (TS 32.240) and Diameter charging protocol (TS 32.299) onto IMS-specific scenarios, CDR content, and message formats.

Key topics:
- IMS charging architectures (§4): offline (Rf→CDF→CGF→BD), online (Ro→OCS), converged (Nchf→CHF)
- IMS charging principles (§5.1): ICID, IOI, SDP handling, trigger conditions, RTTI, roaming, location
- Offline charging scenarios (§5.2): per-node message flows for MO/MT session, mid-session, release, AS-related, SRVCC, OMR
- Online charging scenarios (§5.3): IEC/ECUR/SCUR flows applied to IMS sessions
- Converged charging scenarios (§5.4): CHF-based flows for MRFC and IMS-GWF/SIP AS
- CDR content (§6.1): per-node CDR field definitions (S-CSCF, P-CSCF, I-CSCF, MRFC, MGCF, BGCF, SIP AS, IBCF, E-CSCF, TRF, ATCF, TF)
- Ro message contents (§6.2): IMS-specific Debit/Reserve Units Request/Response
- IMS charging parameters (§6.3): formal IMS Information element and CDR parameter descriptions
- Converged charging data (§6.4): CHF CDR parameters and resources attributes
